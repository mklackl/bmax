## Task: Beta Launch Strategy

Design the beta launch plan:

1. **Beta Type**
   - Closed beta (invite-only) vs open beta vs waitlist
   - Recommended approach with rationale
   - Target beta user count

2. **Invite System** (if closed)
   - Invite code generation strategy
   - Referral mechanics (invite friends for early access?)
   - Waitlist prioritization criteria

3. **Feedback Collection**
   - In-app feedback widget
   - Onboarding survey (3-5 questions max)
   - Exit survey on churn
   - Feedback channels (email, Discord, in-app chat)
   - How to synthesize feedback into actionable items

4. **Success Metrics for Beta**
   - Activation rate target
   - Retention target (Day 1, Day 7, Day 30)
   - NPS or satisfaction score target
   - Number of critical bugs found
   - Feature requests that validate the roadmap

5. **Beta → GA Transition**
   - Exit criteria: when is beta done?
   - Pricing transition (beta pricing → GA pricing)
   - Communication plan for beta users
   - Public launch checklist

Output as `_bmad-output/planning-artifacts/beta-launch.md`.
